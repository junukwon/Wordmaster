# 중학생 영어 단어 집중 학습 웹앱 설계 명세

작성일: 2026-07-10  
대상 콘텐츠: 영어 단어 DAY 01~10, 총 250개(향후 DAY 단위 추가)

개발 프로젝트 기준 경로: `C:\dev\wordmaster`

## 1. 목표

영어 기초가 부족한 중학생 한 명이 iPad mini와 Apple Pencil을 주로 사용하여 하루 90~120분 동안 DAY 5개분, 즉 신규 단어 125개를 집중 학습하도록 돕는다. 단순 노출이 아니라 영어→뜻, 뜻→영어, 철자 회상을 모두 거쳐 당일 숙달 여부를 판단하고 이후 1·3·7·14일 간격으로 다시 확인한다.

성공 기준은 다음과 같다.

- 학생이 별도 설명 없이 홈 화면에서 오늘의 학습을 시작할 수 있다.
- 한 세션에서 125개의 현재 상태를 `확실함`, `헷갈림`, `모름`, `남음`으로 확인할 수 있다.
- 단어별로 영어→뜻, 뜻→영어, 철자 쓰기 단계를 수행할 수 있다.
- 필요한 순간에 영어 발음을 들을 수 있다.
- 정규 학습과 별개로 원하는 범위의 테스트를 수시로 볼 수 있다.
- 브라우저를 닫거나 기기를 재시작한 뒤에도 진도와 복습 일정이 유지된다.
- 새 DAY 데이터가 추가되어도 기존 학습 기록이 보존된다.

## 2. 사용자와 사용 환경

- 사용자: 영어 학습에 어려움을 겪는 중학생 한 명
- 주 기기: iPad mini + Apple Pencil
- 보조 기기: 스마트폰, PC 또는 노트북
- 화면: 가로·세로 반응형이며 iPad 가로 화면에 최적화
- 계정: 초기 버전에는 로그인 없음
- 동기화: 초기 버전에는 기기 간 동기화 없음
- 저장: 해당 기기의 브라우저에 학습 기록 저장
- 확장: 향후 계정과 동기화를 추가할 수 있도록 저장 계층을 UI 및 학습 로직과 분리

## 4. 설계 원칙

1. **회상 우선:** 정답을 읽는 시간보다 정답을 떠올리는 시간을 늘린다.
2. **어려운 단어 집중:** 모든 단어를 동일 횟수로 반복하지 않는다.
3. **단계적 난도:** 인지에서 회상, 회상에서 철자로 진행한다.
4. **정직한 자기 평가:** 손글씨는 기계가 판독하지 않고 학생이 정답과 비교한다.
5. **짧은 반복:** 125개를 한 덩어리로 제시하지 않고 5개 단위로 순환한다.
6. **장기 확인:** 당일에 맞힌 단어도 1·3·7·14일 뒤 다시 확인한다.
7. **낮은 조작 부담:** 첫 화면에서 오늘 할 일을 고민하지 않고 바로 시작한다.

## 5. 학습 모델

### 5.1 단어 숙달 단계

각 단어는 다음 단계를 가진다.

| 단계 | 의미 | 통과 조건 |
|---|---|---|
| `unseen` | 아직 학습하지 않음 | 단어·뜻·발음을 처음 확인 |
| `recognized` | 영어를 보고 뜻을 인지함 | 영어→뜻 문제 통과 |
| `recalled` | 뜻을 보고 영어를 회상함 | 뜻→영어 문제 통과 |
| `spelled` | 철자를 보지 않고 쓸 수 있음 | 필기 후 `기억남` 선택 |
| `mastered_today` | 당일 숙달 | 서로 다른 유형에서 필요한 단계를 모두 통과 |
| `long_term` | 장기 기억 확인 | 14일 복습까지 통과 |

필기 문제는 자동 채점하지 않는다. 학생은 정답을 연 후 자신의 필기와 비교하여 `모름`, `헷갈림`, `기억남` 중 하나를 선택한다.

### 5.2 자기 평가의 효과

| 평가 | 즉시 처리 | 다음 처리 |
|---|---|---|
| 모름 | 현재 단계 유지 또는 한 단계 하락 | 같은 학습 블록 안에서 빠르게 재출제 |
| 헷갈림 | 현재 단계 유지 | 해당 DAY를 마치기 전 재출제 |
| 기억남 | 다음 단계로 이동 | 누적 테스트 또는 예정 복습에서 재확인 |

학생이 `기억남`을 눌러도 한 번의 응답만으로 장기 숙달 처리하지 않는다.

### 5.3 당일 125개 집중 루틴

전체 학습은 약 90~120분을 목표로 한다.

| 구간 | 권장 시간 | 내용 |
|---|---:|---|
| 1부 | 30분 | 125개 빠른 진단, DAY별 5개 묶음 1차 학습 |
| 휴식 | 5~10분 | 앱에 휴식 안내 표시 |
| 2부 | 30분 | 영어→뜻, 뜻→영어 누적 회상과 취약 단어 반복 |
| 휴식 | 5~10분 | 앱에 휴식 안내 표시 |
| 3부 | 30분 | 철자 필기와 125개 전체 혼합 확인 |
| 선택 보강 | 20~30분 | 숙달 기준 미달 단어 집중 복습 |

세션 중 언제든 종료할 수 있고, 다음 실행에서 정확한 위치부터 이어간다.

### 5.4 5개 단위 학습 순환

1. 신규 단어 5개의 영어, 뜻, 품사, 발음을 확인한다.
2. 영어를 보고 뜻을 고른다.
3. 뜻을 보고 영어를 고르거나 떠올린다.
4. 뜻을 보고 Apple Pencil로 철자를 쓴다.
5. 정답을 연 후 `모름`, `헷갈림`, `기억남`으로 평가한다.
6. 모름과 헷갈림은 같은 블록 또는 해당 DAY 안에서 다시 출제한다.
7. 25개를 마치면 DAY 혼합 확인을 실시한다.
8. 새 DAY가 끝날 때 앞에서 학습한 DAY 단어를 일부 섞는다.

### 5.5 장기 간격 복습

당일 숙달 단어의 기본 복습 시점은 학습일 기준 D+1, D+3, D+7, D+14이다.

- `기억남`: 다음 예정 간격으로 이동
- `헷갈림`: 가까운 간격으로 한 단계 축소
- `모름`: 취약 단어에 추가하고 D+1부터 다시 시작
- 예정일이 지났으면 다음 앱 실행 시 오늘의 루틴 앞부분에 우선 배치
- 복습 단어는 당일 신규 125개와 구분해 개수를 표시
- 장기 복습 정답이 신규 125개 진도를 대신 완료하지는 않음

## 6. 발음 기능

- 각 단어의 스피커 버튼을 누를 때만 재생한다.
- 자동 재생은 기본적으로 끈다.
- 초기 구현은 브라우저의 영어 음성 합성 기능을 사용한다.
- 가능한 경우 영어 음성을 우선 선택하고, 재생 속도는 초보자가 듣기 쉬운 수준으로 고정한다.
- 음성 기능을 사용할 수 없으면 학습은 계속 가능하며 버튼 옆에 짧은 안내를 표시한다.
- 한글식 발음 표기는 제공하지 않는다.
- 음성 녹음, 음성 인식, 발음 판정은 하지 않는다.

## 7. Apple Pencil 필기

- 브라우저 Canvas와 Pointer Events를 사용한다.
- 펜, 터치, 마우스를 지원한다.
- 필기 중 캔버스 영역에서는 화면 스크롤을 방지한다.
- 기능은 선 그리기, 되돌리기, 전체 지우기로 제한한다.
- 필기 이미지는 저장하지 않고 다음 문제로 이동할 때 삭제한다.
- 정답을 보기 전에는 영어 철자를 화면에 노출하지 않는다.
- `정답 보기` 후 원래 철자와 학생 필기를 동시에 보고 스스로 평가한다.
- 자동 손글씨 판독과 자동 채점은 하지 않는다.

## 8. 수시 단어 테스트

정규 루틴과 독립적으로 언제든 시작할 수 있다.

### 8.1 설정

- 범위: 특정 DAY 하나, 여러 DAY, 전체
- 단어 집합: 전체, 최근 학습, 헷갈림, 모름, 무작위
- 문제 수: 10, 25, 50, 125
- 유형: 영어→뜻, 뜻→영어, 철자 쓰기, 혼합
- 순서: 번호순, 무작위

### 8.2 결과 처리

- 오답은 즉시 취약 단어에 추가하고 복습 일정에 반영한다.
- 정답을 맞혔다는 이유만으로 예정 복습을 건너뛰지 않는다.
- 테스트 기록은 정규 학습 세션과 별도로 저장한다.
- 테스트를 많이 수행해도 신규 일일 진도가 자동 완료되지 않는다.
- 결과 화면에는 점수, 정답률, DAY별 취약도, 유형별 결과, 오답 목록을 표시한다.
- `틀린 단어만 다시 테스트`와 `틀린 단어만 필기 연습`을 제공한다.

## 9. 화면 구조

### 9.1 홈 — 루틴 중심형

- 오늘의 목표 `125개 단어 도전`
- 전체 진행 막대
- 확실함, 헷갈림, 모름, 남은 단어 수
- 가장 큰 `이어서 학습하기` 버튼
- 보조 동선 `수시 단어 테스트`
- 당일 복습 예정 개수
- 이전 세션이 있으면 새로 시작하지 않고 이어가기 우선

### 9.2 학습 화면

- 상단: DAY 범위, 현재 번호, 125개 진행률
- 좌측 또는 상단: 한국어 뜻, 품사, 발음 버튼
- 우측 또는 하단: Apple Pencil 필기 캔버스
- 필기 도구: 되돌리기, 지우기
- 행동: `모르겠어요`, `정답 보기`
- 정답 확인 후: `모름`, `헷갈림`, `기억남`
- iPad 가로: 문제와 필기장을 좌우 배치
- 세로·스마트폰: 문제와 필기장을 위아래 배치

### 9.3 테스트 설정 화면

- DAY 다중 선택
- 단어 집합 선택
- 문제 유형 선택
- 문제 수 선택
- 출제 순서 선택
- 선택 내용과 예상 소요 시간 요약
- `테스트 시작하기` 버튼

### 9.4 테스트 결과 화면

- 총점과 정답률
- 맞음, 헷갈림, 틀림 개수
- DAY별 결과
- 문제 유형별 결과
- 오답 단어 목록
- 오답 재시험과 필기 연습 버튼
- 홈으로 돌아가기

## 10. 정보 구조와 주요 구성 요소

```text
앱
├─ 홈
│  ├─ 오늘의 루틴
│  ├─ 복습 예정
│  └─ 수시 테스트 진입
├─ 집중 학습
│  ├─ 단어 제시
│  ├─ 발음 재생
│  ├─ 선택형 회상
│  ├─ 필기 캔버스
│  └─ 자기 평가
├─ 단어 테스트
│  ├─ 조건 설정
│  ├─ 문제 진행
│  └─ 결과·오답 재도전
└─ 로컬 학습 기록
   ├─ 단어별 숙달 상태
   ├─ 복습 예정일
   ├─ 진행 중 세션
   └─ 테스트 이력
```

구성 요소는 콘텐츠, 학습 엔진, 저장소, 화면, 기기 기능으로 분리한다. 화면은 학습 엔진의 결과만 표현하고 복습 날짜를 직접 계산하지 않는다. 저장소는 브라우저 저장 방식이 바뀌어도 학습 엔진을 수정하지 않도록 인터페이스로 감싼다.

## 11. 데이터 설계

### 11.1 단어 콘텐츠

```ts
type VocabularyWord = {
  id: string;             // "0001"
  day: number;            // 1
  topic: string;          // "사람 묘사 I"
  term: string;           // "knee"
  partOfSpeech: string[]; // ["명"]
  meanings: string[];     // ["무릎"]
  inflection?: string;    // "grew–grown"
  note?: string;
};
```

콘텐츠는 앱 코드와 분리된 JSON 또는 TypeScript 데이터 파일로 보관한다. 새 사진이나 Markdown이 제공되면 Codex가 내용을 검토하고 다음 DAY 데이터를 추가한다. 앱 안에는 콘텐츠 관리 화면을 만들지 않는다.

type WordProgress = {

## 12. 데이터 흐름

1. 앱 시작 시 콘텐츠 파일과 로컬 학습 기록을 읽는다.
2. 오늘 복습할 단어와 신규 목표 DAY를 계산한다.
3. 홈에 오늘 목표와 이전 진행 상태를 표시한다.
4. 학습 엔진이 현재 단계와 취약도에 따라 다음 문제를 선택한다.
5. 학생의 응답 또는 자기 평가를 학습 엔진에 전달한다.
6. 학습 엔진이 숙달 단계, 취약도, 재출제 위치, 복습 예정일을 계산한다.
7. 저장 계층이 갱신된 기록과 현재 세션 위치를 즉시 저장한다.
8. 화면은 변경된 진행률과 다음 문제를 표시한다.

## 13. 기술 방향

- Codex 구현 작업은 Windows의 `C:\dev\wordmaster`를 프로젝트 루트로 사용한다.
- 프로젝트가 존재하지 않으면 해당 경로에 새 프로젝트를 생성하고, 이미 존재하면 기존 구조와 변경 사항을 먼저 확인한 뒤 작업한다.
- 프런트엔드: React + TypeScript + Vite
- 스타일: 반응형 CSS, CSS Grid/Flexbox
- 저장: 초기에는 `localStorage`를 사용하되 `ProgressRepository` 인터페이스 뒤에 배치
- 필기: HTML Canvas + Pointer Events
- 발음: Web Speech API의 `speechSynthesis`
- 라우팅: 홈, 학습, 테스트 설정, 테스트, 결과의 명시적 화면 상태 또는 경량 라우터
- 배포: 정적 웹사이트로 배포 가능하게 구성
- 테스트: Vitest + Testing Library, 주요 흐름은 Playwright

서버, 데이터베이스, 계정 시스템은 초기 버전에 사용하지 않는다.

## 14. 오류 및 예외 처리

- **음성 없음:** 사용 가능한 영어 음성이 없으면 발음 버튼을 비활성화하고 학습은 계속한다.
- **브라우저 저장 실패:** 즉시 안내하고 현재 세션 데이터를 메모리에 유지한다. 다음 저장 시 재시도한다.
- **저장 데이터 손상:** 손상된 레코드만 기본값으로 복구하고 단어 콘텐츠는 유지한다.
- **콘텐츠 오류:** 앱 시작 시 중복 ID, 누락된 DAY, 빈 단어·뜻을 검사하고 개발 환경에서 명확한 오류를 표시한다.
- **DAY 수 부족:** 선택한 DAY가 5개 미만이면 존재하는 범위만 제안하고 사용자에게 표시한다.
- **복습과 신규 중복:** 동일 단어는 한 세션 큐에서 중복 생성하지 않고 더 높은 우선순위의 목적만 유지한다.
- **중간 종료:** 문제 이동 또는 평가 직후 세션을 저장하여 최대 한 문제만 재진행하게 한다.
- **Canvas 터치:** 필기 영역 밖에서는 정상 스크롤하고 영역 안에서 그리기 중일 때만 스크롤을 막는다.

## 15. 접근성과 사용성

- 주요 터치 영역은 최소 44×44px로 만든다.
- 색상만으로 모름·헷갈림·기억남을 구분하지 않고 글자를 함께 표시한다.
- iPad 가로와 세로, 스마트폰, 데스크톱에서 핵심 기능이 잘리지 않아야 한다.
- 정답 확인 전 실수로 영어 철자가 노출되지 않게 한다.
- 애니메이션은 짧고 기능적인 피드백에만 사용한다.
- 긴 세션에서 피로를 줄이도록 휴식 시점을 명확히 알린다.
- 완료 축하 효과는 제공하되 다음 문제 진행을 방해하지 않는다.

## 19. 완료 정의

초기 버전은 다음 조건을 모두 만족할 때 완료로 본다.

- DAY 01~10, 250개 단어를 오류 없이 불러온다.
- 오늘 학습 DAY 5개와 125개 단어를 선택해 루틴을 시작할 수 있다.
- 5개 단위 학습, DAY 확인, 누적 확인, 필기 회상을 완료할 수 있다.
- 발음 버튼이 지원되는 브라우저에서 영어 발음을 재생한다.
- Apple Pencil로 쓰고 정답과 직접 비교할 수 있다.
- 모름·헷갈림·기억남이 당일 큐와 복습 일정에 반영된다.
- 학습을 중단한 뒤 이어서 진행할 수 있다.
- 원하는 DAY와 조건으로 수시 테스트를 만들 수 있다.
- 테스트 오답이 취약 단어에 반영되며 정규 진도를 임의로 완료하지 않는다.
- 새 DAY 데이터 추가 후에도 기존 진도가 유지된다.
- iPad mini 가로·세로, 스마트폰, PC에서 핵심 흐름이 정상 작동한다.
