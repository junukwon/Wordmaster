# WordMaster DAY 선택 학습 설계

작성일: 2026-07-11

## 1. 배경

WordMaster에는 DAY 01~10의 단어 250개가 모두 들어 있지만, 현재 집중 학습은 DAY 5개를 자동 선택해 125개로 시작한다. 실제 학습 시간은 날마다 달라질 수 있으므로 사용자가 교재의 DAY 구분을 보고 원하는 DAY를 자유롭게 조합해 학습할 수 있어야 한다.

## 2. 목표

- 홈에서 DAY 01~10의 주제와 진행 상태를 한눈에 확인한다.
- 서로 떨어진 DAY를 포함해 1개 이상을 자유롭게 복수 선택한다.
- 선택한 DAY 수에 따라 신규 목표를 25개 단위로 계산한다.
- 진행 중인 세션을 실수로 잃지 않고 이어서 학습하거나 명시적으로 교체한다.
- 기존 필기, 발음, 숙달 판정, D+1·3·7·14 복습, 수시 테스트, 오프라인 저장 동작을 유지한다.

## 3. 범위에서 제외하는 것

- DAY 안에서 개별 단어를 골라 세션을 만드는 기능
- 여러 개의 진행 중 세션을 동시에 저장하는 기능
- 서버 동기화, 로그인, 사용자별 계정
- 기존 복습 간격이나 숙달 판정 규칙 변경
- 원본 250개 단어 콘텐츠 변경

## 4. 홈 화면 구성

### 4.1 진행 중 학습

완료되지 않은 활성 세션이 있으면 홈 상단에 별도 배너를 표시한다.

- 선택된 DAY: 예) `DAY 02 · DAY 07`
- 신규 목표: 예) `50개`
- 현재 진행률
- `이어서 학습하기` 버튼

완료된 세션은 진행 중 배너로 표시하지 않는다.

### 4.2 DAY 카드 그리드

DAY 01~10을 각각 독립된 선택 카드로 표시한다. 각 카드에는 다음 내용을 포함한다.

- DAY 번호
- 원본 교재의 주제
- 단어 수 `25개`
- `숙달`, `학습 중`, `미학습` 개수
- 선택 상태를 나타내는 체크 표시와 시각적 테두리

상태 집계 기준은 다음과 같다.

- `숙달`: 진행 단계가 `mastered_today` 또는 `long_term`인 단어
- `학습 중`: 진행 기록은 있지만 숙달 단계가 아닌 단어
- `미학습`: 진행 기록이 없는 단어

카드 전체를 선택 버튼으로 사용하며 Apple Pencil과 손가락으로 누르기 충분한 크기를 확보한다. iPad 세로 화면에서는 2열을 기본으로 하고, iPad 가로 및 PC에서는 화면 폭에 따라 열 수를 늘린다.

### 4.3 선택 요약과 시작 버튼

DAY 카드는 임의의 조합으로 선택하거나 해제할 수 있다. 선택 순서와 관계없이 화면 표시와 실제 학습 순서는 DAY 번호 오름차순으로 정규화한다.

선택 영역 아래에는 다음 요약을 표시한다.

- 선택한 DAY 수
- 신규 단어 수: `선택 DAY 수 × 25`
- 현재 예정된 복습 단어 수

예: `3일 선택 · 신규 75개 · 복습 9개`

DAY를 선택하지 않으면 시작 버튼을 비활성화한다. 하나 이상 선택하면 `75개 학습 시작하기`처럼 실제 신규 목표 수를 버튼 문구에 반영한다. 홈 진입 시 새 학습을 위한 DAY 선택은 비어 있는 상태로 시작한다.

## 5. 세션 동작

### 5.1 새 세션 생성

홈은 선택된 DAY ID 배열을 세션 생성기에 전달한다. 세션 생성기는 중복을 제거하고 오름차순으로 정렬한 뒤 해당 DAY의 모든 단어를 신규 목표로 삼는다.

- DAY 02·07 선택: 50개
- DAY 01·04·10 선택: 75개
- DAY 01~10 전체 선택: 250개

기존 영어→뜻, 뜻→영어, 철자 필기와 어려운 단어 재출제 순환은 선택된 단어 집합에 그대로 적용한다. 학습 화면의 진행률과 접근성 레이블은 고정된 `125개`가 아니라 현재 세션의 `targetWordIds.length`를 사용한다.

### 5.2 예정 복습

예정일이 된 복습 단어는 기존과 같이 신규 학습보다 먼저 배치한다. 복습 단어 수는 신규 목표 수와 분리해 표시하며 신규 진도를 대신 완료하지 않는다. 선택된 신규 목표에도 포함된 단어는 중복 복습 항목으로 앞에 추가하지 않는다.

### 5.3 진행 중 세션 보호

완료되지 않은 활성 세션이 있을 때 사용자가 다른 DAY 조합으로 새 학습을 시작하면 확인 대화상자를 표시한다.

- `기존 학습 이어하기`: 대화상자를 닫고 기존 세션으로 이동한다.
- `새 학습으로 교체`: 활성 세션만 새 조합으로 교체한다.
- 대화상자 닫기 또는 취소: 기존 세션과 현재 DAY 선택을 모두 그대로 둔다.

세션을 교체해도 단어별 학습 결과, 자신감, 복습 예정일과 수시 테스트 기록은 삭제하지 않는다. 완료된 세션만 남아 있다면 확인 없이 새 세션을 만들 수 있다.

## 6. 컴포넌트와 데이터 경계

- `App`: 저장소에서 단어별 진행과 활성 세션을 읽고, DAY별 요약 데이터를 만든다. 사용자가 확정한 DAY 배열로 새 세션을 생성한다.
- `HomePage`: DAY 선택 상태, 카드 렌더링, 선택 합계, 활성 세션 배너와 교체 확인 UI를 담당한다. 영구 학습 기록을 직접 변경하지 않는다.
- `DaySelectionGrid` 또는 동등한 전용 컴포넌트: DAY 카드 목록과 선택/해제를 담당한다. DAY ID 배열을 입력·출력하는 단순 경계를 갖는다.
- `sessionEngine`: 전달받은 임의 DAY 조합을 검증·정규화하고 정확한 목표 단어와 학습 큐를 생성한다.
- `LocalStorageProgressRepository`: 기존 저장 형식을 유지한다. 새 영구 저장 키는 필요하지 않으며 활성 세션 교체는 기존 저장 메서드를 사용한다.

선택 상태는 아직 세션이 아니므로 홈의 메모리 상태로만 유지한다. 새 세션 생성이 확정된 뒤에만 저장소를 변경한다.

## 7. 오류 및 접근성

- 빈 DAY 배열로는 세션을 만들지 않는다.
- 존재하지 않는 DAY ID는 세션 생성 단계에서 제외하며, 유효한 DAY가 하나도 남지 않으면 저장소를 변경하지 않고 사용자에게 오류를 알린다.
- 저장 실패 시 현재 활성 세션을 보존하고 기존 저장 오류 안내 영역을 사용한다.
- 선택 카드는 실제 버튼 또는 동등한 키보드 조작 요소로 구현한다.
- 선택 상태는 색상만으로 표현하지 않고 체크 표시와 `aria-pressed`를 함께 사용한다.
- 교체 확인 UI는 포커스 이동, Escape 취소, 명확한 버튼 이름을 제공하는 접근 가능한 대화상자로 구현한다.
- 진행률 변경은 현재 세션 크기를 포함한 텍스트와 접근성 레이블로 전달한다.

## 8. 오프라인과 PWA

DAY 카드와 선택 로직은 기존 번들에 포함되며 추가 네트워크 요청에 의존하지 않는다. 단어 데이터와 학습 기록은 기존 로컬 우선 구조를 사용한다. 최초 온라인 접속 후에는 오프라인에서도 DAY 선택, 새 세션 생성, Apple Pencil 필기, 진도 저장과 수시 테스트가 동작해야 한다. 앱 업데이트는 서비스 워커 캐시만 교체하고 기존 학습 기록은 유지한다.

## 9. 테스트 계획

### 9.1 도메인 테스트

- DAY 02·07로 정확히 50개의 중복 없는 목표를 생성한다.
- DAY 01·04·10으로 정확히 75개의 중복 없는 목표를 생성한다.
- 입력 순서와 중복에 관계없이 DAY를 오름차순으로 정규화한다.
- 빈 배열과 유효하지 않은 DAY만 있는 배열로 세션을 만들지 않는다.
- 복습 단어가 신규 목표에 중복 삽입되지 않는다.

### 9.2 화면 테스트

- DAY 01~10 카드에 번호, 주제, 25개와 상태 집계가 표시된다.
- 카드 선택과 해제로 DAY 수 및 신규 단어 합계가 갱신된다.
- 미선택 상태에서는 시작 버튼이 비활성화된다.
- 진행 중 세션 배너가 정확한 DAY와 진도를 표시한다.
- 교체 확인의 이어하기, 교체, 취소가 각각 올바르게 동작한다.
- 학습 화면의 `125개` 고정 문구가 현재 세션 크기로 바뀐다.

### 9.3 E2E 테스트

- 임의의 DAY 두 개를 선택해 50개 세션을 시작하고 첫 문제를 학습한다.
- 진행 중 세션을 보존한 채 홈으로 돌아와 이어서 학습한다.
- 다른 DAY를 선택한 뒤 교체를 취소하면 기존 세션이 유지된다.
- 교체를 확인하면 새 DAY 조합으로 시작하고 기존 단어별 진행 기록은 유지된다.
- iPad Mini 세로·가로 초기 화면에서 카드와 시작 버튼을 사용할 수 있다.
- 최초 방문 후 오프라인 상태에서 DAY 선택, 학습 시작, 필기와 진도 저장이 동작한다.

## 10. 완료 기준

- 사용자가 DAY 01~10 중 임의의 DAY를 1개 이상 자유롭게 선택할 수 있다.
- 선택한 DAY의 모든 단어만 신규 목표에 포함되며 목표 수가 정확하다.
- DAY별 진행 상태와 전체 선택 합계가 홈에서 명확하다.
- 진행 중 세션이 명시적 확인 없이 교체되지 않는다.
- 기존 학습 기록, 복습 일정, 필기, 발음, 수시 테스트와 오프라인 기능이 회귀하지 않는다.
- 단위 테스트, 프로덕션 빌드와 E2E 테스트가 통과한다.
