# WordMaster 학습 범위 설정 홈 통합 설계

## 목적

학습 범위 설정 화면을 기존 홈과 자연스럽게 연결한다. DAY 카드 기반의 빠른 학습은 유지하면서, 묶음·범위·랜덤 학습을 홈에서 언제든 찾을 수 있게 한다.

## 선택한 접근

기존 홈을 대시보드로 유지하고 홈 상단의 주요 학습 영역에 `학습 범위 설정` 진입 버튼을 항상 표시한다.

- 기존 DAY 카드 직접 선택은 빠른 학습 방식으로 유지한다.
- 묶음·범위·랜덤 선택은 `/study/setup` 전용 화면에서 제공한다.
- 진행 중인 학습이 있어도 설정 화면 진입 버튼을 숨기지 않는다.
- 진행 중인 학습 교체 확인은 설정 화면에 들어갈 때가 아니라 새 범위로 학습을 시작할 때만 표시한다.

## 화면 및 이동 구조

### 홈 (`/`)

- 진행 중인 세션이 있으면 `이어서 학습하기`를 표시한다.
- `학습 범위 설정` 버튼은 세션 유무와 관계없이 항상 표시한다.
- DAY 카드 선택과 `수시 단어 테스트`는 현재 동작을 유지한다.
- DAY가 늘어나도 범위 설정 진입 버튼이 카드 목록 아래에 묻히지 않도록 DAY 목록보다 앞쪽에 배치한다.

### 학습 범위 설정 (`/study/setup`)

- 묶음, 범위, 랜덤 DAY, 랜덤 단어 선택 방식을 유지한다.
- 홈으로 나갔다 돌아와도 마지막으로 선택한 탭과 입력값을 복원한다.
- 선택값 복원은 앱 실행 중에만 유지하는 `sessionStorage`를 사용한다.
- 랜덤 선택의 실제 결과는 현재와 같이 학습을 시작하는 시점의 세션에 고정한다.
- 진행 중인 학습이 있으면 `선택한 범위로 학습 시작`을 누른 뒤 교체 확인 대화상자를 표시한다.

## 상태와 저장

- 학습 진도와 진행 중 세션은 기존 `localStorage` 키 `wordmaster:v1`을 그대로 사용한다.
- 설정 화면의 임시 선택은 별도 `sessionStorage` 키를 사용하며 학습 진도 데이터에 섞지 않는다.
- 저장할 값은 선택 방식, 묶음 시작 DAY, 범위 시작·종료 DAY, 랜덤 방식, DAY 수, 단어 수, 검색어다.
- 임시 값이 없거나 유효하지 않으면 현재 기본값으로 안전하게 돌아간다.
- 임시 선택 저장 실패가 학습 화면 진입이나 기존 진도 읽기를 막아서는 안 된다.

## 구성 요소 경계

- `HomePage`는 상시 진입 링크의 위치와 표시만 담당한다.
- `StudySetupPage`는 설정값을 읽고 각 선택 컨트롤에 전달하며 변경 시 임시 저장한다.
- 설정 직렬화와 유효성 검사는 작은 저장소 또는 순수 헬퍼로 분리해 단독 테스트할 수 있게 한다.
- `AppRouter`의 기존 라우트와 실제 학습 세션 생성 흐름은 변경하지 않는다.

## 테스트

- 진행 중인 세션이 있어도 홈 상단에서 `학습 범위 설정` 링크가 보인다.
- 진행 중인 세션이 없을 때도 같은 링크가 보이며 중복 렌더링되지 않는다.
- 설정 화면에서 선택 방식을 바꾼 뒤 홈으로 이동했다 돌아오면 선택값이 복원된다.
- 손상되거나 범위를 벗어난 임시 저장값은 기본값으로 대체된다.
- 진행 중인 세션은 설정 화면에 들어가는 것만으로 교체되지 않는다.
- 새 범위의 시작 버튼을 누를 때만 기존 교체 확인 대화상자가 나타난다.
- 전체 단위 테스트와 빌드, E2E 테스트로 기존 학습·테스트·PWA 동작을 회귀 검증한다.

## 제외 범위

- 홈의 DAY 카드 제거 또는 무한 스크롤 도입
- 서버 동기화나 계정 기반 저장
- 브라우저를 닫은 뒤에도 임시 설정값을 영구 보존
- 학습 세션 및 진도 저장 형식 변경
