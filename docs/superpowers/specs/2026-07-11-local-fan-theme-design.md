# WordMaster 로컬 팬 테마 이미지팩 설계

작성일: 2026-07-11

## 1. 배경

WordMaster를 사용하는 학생이 세븐틴 팬이라는 점을 학습 분위기에 자연스럽게 반영한다. 이미지 자체를 정답 보상이나 별도 감상 콘텐츠로 만들면 학습 평가를 왜곡하거나 이미지 화면에만 머물 가능성이 있으므로, 팬 이미지는 앱 전반의 비상호작용 테마 요소로만 사용한다.

현재 개인 이미지 폴더 `C:\dev\wordmaster\seventeen`에는 정적 이미지 142장, 약 87MB가 있다. 형식은 PNG 120장, JPG 21장, WebP 1장이며 정사각형에 가까운 이미지 90장, 가로 이미지 50장, 세로 이미지 1장이다. 자막이 이미지 가장자리에 있는 경우가 많으므로 강제 중앙 크롭을 피해야 한다.

## 2. 목표

- 팬 이미지를 홈, 학습, 결과 화면에 자연스럽게 배치한다.
- 정답·오답·숙달 평가와 이미지 노출을 완전히 분리한다.
- 이미지팩을 공개 GitHub 저장소, Pages 배포물, 서비스워커 선캐시에 포함하지 않는다.
- iPad에서 개인 이미지팩을 한 번 가져오면 오프라인에서도 표시한다.
- 원본 약 87MB를 그대로 복제하지 않고 기기용 이미지로 축소·압축한다.
- 이미지팩이 없거나 실패해도 기존 단어 학습 기능은 그대로 작동한다.

## 3. 범위에서 제외하는 것

- 정답 수, `기억남`, 숙달 단계에 따른 이미지 해금이나 보상
- 이미지 감상 메뉴, 앨범, 확대 보기, 다운로드, 공유
- 이미지 클릭이나 사진별 반응 기능
- 서버 업로드, 계정 동기화, 다른 기기와의 이미지팩 동기화
- GitHub Pages에 팬 이미지 배포
- MP4, MOV, GIF, 애니메이션 WebP 지원
- 얼굴 인식, 인물 분류, 자동 자막 분석
- 앱이 이미지의 저작권 상태를 판정하는 기능

## 4. 사용자 경험

### 4.1 테마 설정

설정 영역에 다음 조작만 제공한다.

- `팬 테마 이미지 가져오기`: JPG, JPEG, PNG, 정적 WebP 다중 선택
- `팬 테마 사용`: 이미지팩을 유지한 채 테마 표시 켜기·끄기
- `이미지팩 삭제`: 확인 후 로컬 이미지팩만 삭제

별도 이미지 목록이나 감상 화면은 제공하지 않는다. 이미지팩을 다시 가져오면 기존 팩에 추가하지 않고 새 팩 전체로 교체한다.

### 4.2 화면 배치

- 홈: 상단 테마 영역에 오늘의 대표 이미지 1장을 비교적 크게 표시한다.
- DAY 선택 카드: 사진을 넣지 않고 팬 테마의 파스텔 포인트 색상만 반영한다.
- 집중 학습: 문제·정답·Apple Pencil 필기장과 분리된 작은 응원 이미지를 표시한다. 5단어 학습 블록이 바뀔 때만 이미지를 교체한다.
- 테스트 실행: 문제 풀이 중에는 이미지를 표시하지 않는다.
- 학습 완료 및 테스트 결과: 결과 내용과 분리해 이미지 1장을 크게 표시한다.

필기장과 문제 본문에는 이미지 배경을 사용하지 않는다. 팬 테마가 꺼져 있거나 이미지팩이 없으면 현재 WordMaster UI를 그대로 렌더링한다.

### 4.3 이미지 프레임

원본 전체를 `object-fit: contain`으로 표시해 얼굴과 자막을 보존한다. 프레임에 남는 여백은 같은 이미지를 확대한 뒤 흐림·어둡기 처리를 한 배경 레이어로 채운다. 배경 레이어는 장식용이며 접근성 트리에서 제외한다.

## 5. 이미지 선택 규칙

이미지 선택은 학습 평가와 독립적인 결정적 함수로 만든다.

- 홈: 로컬 날짜를 기준으로 하루 동안 동일한 이미지
- 학습: 세션 ID와 5단어 블록 ID를 기준으로 동일한 이미지
- 학습 완료: 세션 ID를 기준으로 동일한 이미지
- 테스트 결과: 테스트 시도 ID를 기준으로 동일한 이미지

같은 날짜·세션·블록·시도를 다시 열면 같은 이미지가 나온다. 정답, 오답, 자신감 등급, 복습 단계는 이미지 선택 키에 포함하지 않는다. 이미지가 1장뿐이어도 정상 동작하며, 이미지팩 교체 후에는 새 팩의 개수에 맞춰 다시 선택한다.

## 6. 가져오기와 최적화

### 6.1 입력 제한

- 허용 형식: `image/jpeg`, `image/png`, 정적 `image/webp`
- 최대 이미지 수: 150장
- 변환 후 이미지팩 최대 합계: 30MB
- 움직이는 이미지 또는 디코딩할 수 없는 파일은 제외하고 결과에 개수를 표시한다.

### 6.2 순차 처리

브라우저는 파일을 한 장씩 디코딩한다. 동시에 여러 원본을 비트맵으로 펼치지 않는다.

각 파일은 다음 순서로 처리한다.

1. MIME 형식과 정적 이미지 여부를 확인한다.
2. 이미지 방향 정보를 반영해 디코딩한다.
3. 종횡비를 유지하며 긴 변을 최대 720px로 축소한다. 720px보다 작은 원본은 확대하지 않는다.
4. WebP 품질 0.70으로 인코딩한다.
5. WebP 인코딩을 지원하지 않으면 JPEG 품질 0.72로 대체한다.
6. 결과 Blob 크기와 누적 크기를 확인한다.
7. 디코딩에 사용한 비트맵·오브젝트 URL·캔버스를 즉시 해제한다.

개수나 30MB 제한을 넘으면 새 팩 저장을 중단하고 기존 팩을 유지한다.

### 6.3 원자적 교체

새 팩은 IndexedDB 임시 저장 영역에 먼저 기록한다. 모든 허용 파일 처리와 메타데이터 기록이 성공한 뒤 활성 팩 포인터를 한 번에 새 팩으로 바꾸고 이전 팩을 삭제한다.

변환 실패, IndexedDB 오류, 저장 공간 부족, 앱 종료가 발생하면 활성 팩 포인터를 변경하지 않는다. 다음 실행에서 완료되지 않은 임시 팩을 정리한다.

## 7. 저장 구조

이미지 Blob과 메타데이터는 별도 IndexedDB 데이터베이스 `wordmaster-fan-theme-v1`에 저장한다. 기존 `localStorage` 키 `wordmaster:v1`과 섞지 않는다.

저장 단위는 다음 책임을 가진다.

- 팩 메타데이터: 팩 ID, 이미지 수, 합계 크기, 생성 시각, 인코딩 형식
- 이미지: 팩 ID, 순번, Blob, 너비, 높이
- 설정: 활성 팩 ID, 팬 테마 사용 여부
- 임시 팩: 가져오기 중인 팩과 완료 상태

이미지 URL은 렌더링할 때 Blob으로부터 만들고 컴포넌트가 바뀌거나 언마운트될 때 해제한다. 학습 기록 삭제와 이미지팩 삭제는 별도 동작이다.

## 8. PWA, 오프라인과 업데이트

- 팬 이미지는 Vite 번들, `public/`, Git 저장소와 Workbox `globPatterns`에 넣지 않는다.
- 팬 이미지를 위한 네트워크 요청이나 런타임 HTTP 캐시는 만들지 않는다.
- 최초 가져오기 이후 IndexedDB에서만 읽으므로 오프라인에서도 표시된다.
- 서비스워커 및 앱 버전 업데이트는 IndexedDB를 삭제하거나 마이그레이션하지 않는다.
- 저장공간 정책에 의해 Safari가 사이트 데이터를 제거하면 이미지팩도 사라질 수 있음을 설정 화면에 안내한다.
- 이미지팩이 사라져도 기존 앱, 단어 데이터와 가능한 학습 기록을 정상적으로 로드한다.

## 9. 오류 처리

- 지원하지 않는 파일: 해당 파일을 제외하고 처리 결과에 표시한다.
- 손상 파일: 해당 파일을 제외하고 파일명과 실패 개수를 표시한다.
- 150장 초과 또는 30MB 초과: 새 팩 전체를 적용하지 않고 기존 팩을 유지한다.
- 저장 공간 부족·IndexedDB 트랜잭션 실패: 임시 팩을 폐기하고 기존 팩을 유지한다.
- 이미지 렌더링 실패: 해당 프레임만 기본 팬 테마 그라데이션으로 대체한다.
- 설정 데이터 손상: 이미지팩 사용을 끄고 학습 화면을 기본 UI로 표시한다.

오류 메시지는 학습을 막는 전역 장애로 처리하지 않는다. 이미지팩 가져오기 화면에 결과를 표시하고 단어 학습은 계속할 수 있어야 한다.

## 10. 접근성 및 성능

- 이미지에는 학습 내용으로 오해되지 않도록 빈 대체 텍스트를 사용하거나 장식 이미지 역할을 부여한다.
- 팬 이미지는 단어, 뜻, 진행률, 버튼의 접근 가능한 이름에 포함하지 않는다.
- 포인트 색상은 기존 대비 기준과 `:focus-visible` 표시를 유지한다.
- `prefers-reduced-motion`에서는 이미지 전환 애니메이션을 사용하지 않는다.
- 현재 화면에 필요한 1장만 Blob URL로 유지한다.
- 다음 이미지를 미리 디코딩하더라도 최대 1장까지만 허용한다.
- 가져오기 진행률은 처리 장수로 표시하며 메인 스레드를 장시간 독점하지 않도록 각 이미지 사이에 제어권을 반환한다.

## 11. 개인정보와 저작권

팬 이미지는 사용자의 기기 내부에서만 처리·저장한다. 앱은 이미지를 GitHub, 외부 서버, 분석 서비스로 전송하지 않는다. 공개 저장소에는 샘플 팬 이미지를 커밋하지 않으며 테스트에는 생성된 단색·그라데이션 이미지 Blob만 사용한다.

개인 보유 이미지의 사용 권한과 저작권 책임은 사용자에게 있음을 설정 화면에 간단히 안내한다.

## 12. 테스트 계획

### 12.1 가져오기 및 저장

- JPG, PNG, 정적 WebP를 순차적으로 변환한다.
- 긴 변 720px 제한과 종횡비를 유지한다.
- 작은 원본을 확대하지 않는다.
- WebP 미지원 환경에서 JPEG로 대체한다.
- 150장 및 변환 결과 30MB 제한을 적용한다.
- 손상 파일을 제외하고 결과 개수를 표시한다.
- 일부 변환 실패, 저장 공간 부족, 트랜잭션 실패 때 기존 팩을 유지한다.
- 성공한 새 팩만 원자적으로 활성화하고 이전 팩을 삭제한다.
- 미완료 임시 팩을 다음 실행에서 정리한다.

### 12.2 선택 규칙

- 같은 날짜는 같은 홈 이미지를 선택한다.
- 같은 세션·블록은 같은 학습 이미지를 선택한다.
- 같은 세션·테스트 시도는 같은 결과 이미지를 선택한다.
- 정답·오답·평가 변경은 선택 이미지에 영향을 주지 않는다.
- 0장, 1장, 142장 팩에서 안전하게 동작한다.

### 12.3 화면 및 접근성

- 이미지팩이 없거나 테마가 꺼져 있으면 현재 UI와 동일하다.
- 홈, 학습, 완료, 테스트 결과의 지정 위치에만 이미지가 나타난다.
- 테스트 실행, 문제 본문, 필기장에는 이미지가 나타나지 않는다.
- `contain` 이미지와 흐린 배경이 가로·정사각·세로 비율에서 자막을 자르지 않는다.
- 팬 이미지가 접근성 트리의 학습 텍스트를 오염시키지 않는다.
- iPad Mini 세로·가로에서 버튼과 필기 공간이 기존 크기를 유지한다.
- Apple Pencil 필기 지연과 팜리젝션이 회귀하지 않는다.

### 12.4 PWA 및 오프라인

- 팬 이미지가 빌드 산출물과 서비스워커 선캐시에 포함되지 않는다.
- 네트워크를 끈 뒤 홈·학습·결과 이미지가 IndexedDB에서 표시된다.
- 앱 캐시 업데이트 후 학습 기록과 이미지팩이 유지된다.
- 이미지 DB가 없거나 제거된 상태에서도 오프라인 단어 학습을 시작할 수 있다.

## 13. 완료 기준

- 사용자가 iPad에서 최대 150장의 정적 이미지를 가져올 수 있다.
- 변환된 팩은 30MB 이내이며 원본을 앱 저장소에 복제하지 않는다.
- 팬 이미지는 보상·앨범 없이 지정된 UI 위치에만 표시된다.
- 이미지 선택은 학습 평가와 독립적이고 같은 컨텍스트에서 안정적이다.
- 팬 이미지가 GitHub Pages, Git, 서비스워커 캐시에 포함되지 않는다.
- 가져오기 실패 또는 이미지팩 부재가 단어 학습을 중단시키지 않는다.
- 기존 DAY 선택, 복습, 수시 테스트, 발음, Apple Pencil, PWA와 오프라인 테스트가 통과한다.
