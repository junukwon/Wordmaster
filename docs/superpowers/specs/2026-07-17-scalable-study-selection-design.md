# WordMaster 확장형 학습 선택과 랜덤 학습 설계

작성일: 2026-07-17
상태: 설계 승인 후 구현 계획 작성 전

## 1. 배경과 목표

WordMaster는 현재 DAY 단위로 학습 범위를 고르는 구조다. DAY 11~20 단어장이 추가되고 앞으로 100 DAY(약 2,500단어)까지 늘어날 수 있으므로, 첫 화면에 DAY 카드 100개를 한 번에 나열하지 않으면서도 원하는 범위를 빠르게 시작할 수 있어야 한다.

이번 변경의 목표는 다음과 같다.

- DAY 01~20 및 향후 더 많은 DAY를 같은 UI 규칙으로 제공한다.
- 5일 묶음, 임의 DAY 범위, 전체 단어 중 랜덤 단어 세트를 모두 지원한다.
- 랜덤으로 고른 결과는 한 세션 동안 고정해 공정한 복습과 재개를 보장한다.
- 기존 진도, D+1·3·7·14 복습, 수시 테스트, 필기와 오프라인 동작을 훼손하지 않는다.
- 단어 수가 늘어도 정적 데이터와 기존 localStorage 학습 기록을 그대로 사용할 수 있게 한다.

범위 밖: 로그인·서버·손글씨 자동 인식·음성 인식·발음 평가, 별도의 보상/팬 이미지 기능 추가.

## 2. 사용자 흐름

### 2.1 학습 범위 선택

시작 화면의 선택 영역은 세 가지 모드 탭으로 구성한다.

1. **묶음으로 선택**: 5 DAY를 한 묶음으로 보여준다. 예: `DAY 01–05`, `DAY 06–10`, `DAY 11–15`, `DAY 16–20`. 각 묶음에는 포함 단어 수와 진행 요약을 표시한다.
2. **범위로 선택**: 시작 DAY와 끝 DAY를 고른다. 두 값은 자동으로 정규화해 오름차순 범위를 만든다. 범위 안의 모든 단어가 대상이다.
3. **랜덤으로 선택**: 아래 두 가지 중 하나를 고른다.
   - `랜덤 DAY 묶음`: 전체 DAY에서 서로 다른 DAY를 지정 개수만큼 무작위로 뽑고, 뽑힌 DAY의 모든 단어를 대상에 넣는다.
   - `랜덤 단어 세트`: 전체 어휘에서 중복 없이 단어를 무작위로 뽑는다. 선택 수 프리셋은 10·25·50·125이며 기본값은 25다.

초기 화면은 묶음 목록을 우선 보여주고, DAY가 많아질 때를 대비해 DAY 번호/주제 검색과 범위 입력을 함께 제공한다. 100 DAY에서도 화면은 20개 묶음으로 유지되며, 특정 DAY로 바로 이동할 수 있다.

### 2.2 시작 및 세션 고정

사용자가 시작 버튼을 누르면 선택 조건을 실제 `targetDayIds`와 `targetWordIds`로 해석한다. 랜덤 모드는 이 순간 한 번만 난수를 사용한다. 생성된 ID 배열은 세션에 저장하고 학습 화면에서는 다시 계산하지 않는다.

- 선택 결과가 비어 있으면 시작 버튼을 비활성화한다.
- 같은 단어 ID는 한 세션에서 한 번만 등장한다.
- 진행 중인 세션이 있으면 기존 세션을 이어가거나 새 선택으로 교체할지 확인한다.
- 세션을 재개하면 저장된 랜덤 결과를 그대로 사용한다.

### 2.3 복습과의 관계

`오늘 복습`은 선택 모드와 별도의 진입점으로 유지한다. 사용자가 랜덤 학습을 고르더라도 D+1·3·7·14 예정 단어를 자동으로 섞지 않는다. 기존 학습 화면의 필기, 발음 듣기, 정답 보기/기억남 흐름과 수시 테스트는 대상 단어 ID만 바뀌고 동작은 그대로다.

## 3. 데이터와 도메인 설계

### 3.1 DAY 카탈로그

콘텐츠 빌드 결과에서 DAY별 메타데이터를 계산한다.

```ts
type DaySummary = {
  dayId: string;          // "DAY01"
  dayNumber: number;      // 1
  title?: string;
  wordCount: number;
  masteredCount: number;
  learningCount: number;
  unstartedCount: number;
};
```

화면은 `DaySummary[]`만 사용하고, 단어 원문 배열을 매번 직접 그룹화하지 않는다. 콘텐츠가 추가되면 카탈로그가 자동으로 DAY를 늘린다.

### 3.2 선택 명령

선택 UI와 세션 엔진 사이에는 명시적인 명령 타입을 둔다.

```ts
type StudySelection =
  | { mode: 'bundle'; bundleStartDay: number }
  | { mode: 'range'; startDay: number; endDay: number }
  | { mode: 'random-days'; dayCount: number; seed?: string }
  | { mode: 'random-words'; wordCount: number; seed?: string };
```

엔진은 명령을 검증한 뒤 결정론적인 결과 객체를 반환한다.

```ts
type StudyTarget = {
  targetDayIds: string[];
  targetWordIds: string[];
  selection: StudySelection;
};
```

난수 생성기는 주입 가능하게 만들어 단위 테스트에서 고정 seed 또는 가짜 RNG를 사용한다. 실제 앱에서는 암호학적 난수가 필요하지 않으며 균등한 Fisher–Yates 셔플이면 충분하다.

### 3.3 콘텐츠 확장 규칙

- DAY 01~20 콘텐츠를 기존 정적 JSON 빌드 파이프라인에 포함한다.
- 단어 ID는 기존 ID와 겹치지 않도록 전역 유일성을 검사한다.
- DAY별 단어 수와 전체 단어 수를 빌드 시 검증한다.
- 새 데이터가 추가되어도 기존 localStorage 키와 단어 ID의 의미를 바꾸지 않는다.

## 4. UI 구성

- `StudySetupPage`가 모드 탭, 묶음 목록, 범위 입력, 랜덤 설정을 소유한다.
- `DayBundleList`는 5일 단위 묶음과 진행 요약을 렌더링한다.
- `DayRangePicker`는 유효한 시작/끝 DAY만 허용하고 접근 가능한 레이블을 제공한다.
- `RandomStudyPicker`는 랜덤 DAY 묶음/랜덤 단어 세트와 개수 프리셋을 제공한다.
- 시작 전 요약은 `DAY 11–15 · 125단어` 또는 `전체 중 랜덤 25단어`처럼 대상과 개수를 명확히 표시한다.
- `aria-pressed`, 키보드 포커스, iPad 세로 화면에서의 한 손/Apple Pencil 터치 영역을 기존 기준과 동일하게 유지한다.

100 DAY까지 늘어날 때의 성능을 위해 목록은 묶음 단위로 렌더링하며, 필요하면 검색 결과만 표시한다. 가상 스크롤은 실제 측정에서 병목이 확인될 때만 추가한다.

## 5. 오류 및 경계 조건

- 존재하지 않는 DAY, 음수/0인 개수, 전체 단어 수보다 큰 랜덤 개수는 엔진에서 거부하고 사용자에게 안내한다.
- DAY에 단어가 하나도 없으면 카탈로그에서 선택 불가로 표시한다.
- 콘텐츠 빌드 검증 실패 시 앱 번들 생성을 중단하고 오류 원인을 출력한다.
- localStorage에 저장된 진행 기록이 일부 손상되어도 선택 화면은 열리고, 해당 단어만 미시작 상태로 안전하게 취급한다.
- 오프라인에서는 이미 캐시된 콘텐츠와 앱 셸로 모든 선택 모드와 세션 저장이 동작해야 한다.

## 6. 테스트 전략

### 도메인 단위 테스트

- 5일 묶음 계산이 DAY 수 20 및 100에서 정확하다.
- 범위 선택이 입력 순서를 정규화하고 경계 DAY를 포함한다.
- 랜덤 DAY/랜덤 단어가 요구 개수, 중복 없음, 전체 범위 제한을 만족한다.
- 주입한 RNG/seed로 같은 입력이 같은 결과를 재현한다.
- 세션 재개 시 랜덤 결과와 순서가 변하지 않는다.
- 잘못된 DAY와 개수는 명확한 오류를 반환한다.

### 컴포넌트/E2E 테스트

- 세 모드 전환과 선택 요약/시작 버튼 상태가 갱신된다.
- DAY 11~20 콘텐츠가 카드/범위 선택에 표시된다.
- 랜덤 결과를 다시 뽑은 뒤 시작하면 새 결과가 세션에 고정된다.
- 진행 중 세션을 재개해도 대상 단어 수와 진도가 유지된다.
- 오프라인 상태에서 시작, 학습, 필기, 진도 저장, 수시 테스트가 동작한다.
- iPad mini 세로 화면에서 선택 컨트롤과 시작 버튼이 스크롤 없이 접근 가능하다.

## 7. 완료 기준

- DAY 01~20을 묶음/범위/랜덤 DAY/랜덤 단어 모드로 시작할 수 있다.
- 랜덤 단어 세트의 기본값은 25개이며 10·50·125개로 바꿀 수 있다.
- 100 DAY 확장 시에도 첫 화면의 선택 항목이 묶음 단위로 유지된다.
- 세션 재개, 기존 진도와 복습 일정, 오프라인 기능이 회귀하지 않는다.
- `npm run content:build`, `npm test`, `npm run build`, `npm run test:e2e`가 모두 통과한다.
